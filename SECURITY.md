# 安全政策

## 隐私保护承诺

本项目高度重视用户隐私和数据安全，严格遵循以下隐私保护原则：

### 数据处理方式
1. **本地处理**：所有简历分析均在用户的浏览器本地完成
2. **不上传原文**：绝不上传用户的原始简历或职位描述文本
3. **匿名统计**：仅在用户主动开启统计开关后，才上传匿名统计特征
4. **本地存储**：用户数据优先存储在浏览器本地

### 数据类型
- **绝不上报**：原始简历文本、职位描述文本、个人信息
- **可能上报**（仅在用户授权后）：技能标签、学历等级、经验年限区间、匹配分数、缺失技能等匿名特征

## 安全措施

### 客户端安全
- 所有文本处理在浏览器中完成，不经过服务器
- 使用UUIDv4生成匿名ID，不包含个人信息
- 提供数据清除功能，可删除所有本地存储的项目数据

### 服务端安全
- 所有API端点验证请求内容，仅接受预定义的字段
- 严格的字段白名单控制，防止意外上传敏感信息
- 管理端点需要认证令牌

### 遥测控制
- 遥测功能默认关闭
- 用户可以随时开启或关闭遥测
- 提供隐私设置界面，让用户完全控制数据共享

## 报告安全漏洞

如果您发现任何安全漏洞或隐私问题，请通过以下方式联系：

1. 创建GitHub Issue（如果公开披露不影响安全性）
2. 或者发送邮件至 [您的邮箱] （如果是严重的安全问题）

请在报告中包含：
- 漏洞的详细描述
- 复现步骤
- 潜在影响
- 可能的解决方案

## 安全审计

项目代码定期接受安全审计，重点关注：
- 数据泄露风险
- 未经用户同意的数据传输
- 潜在的隐私侵犯
- 安全漏洞

## 合规性

本项目致力于遵守相关的隐私法律和法规，包括但不限于GDPR（欧盟通用数据保护条例）和CCPA（加州消费者隐私法）。

## 更新

此安全政策可能会定期更新，以反映功能变化或监管要求。重大更新将通过项目发布说明通知用户。
